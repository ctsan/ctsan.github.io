Counting the number of elements in Sequences made simple

2017-08-13
Math
570 words 3 mins read

Contents

  • Example 1: {5,6,7,…,100}
  • Example 2: {36,39,42,…,141}
  • Example 3: {87.3, 86.2, 85.1, …, -7.3}

Counting the number of elements in a sequence of elements can often be confusing, and this is something that frequently comes up in mathematics, computer science or even programming (I won’t get deeper in the use cases in this post). Thankfully there is a simple method that I am using in such scenarios which makes these mini problems straightforward. Only basic mathematics are required.

Examples First

Let’s start in an unorthodox fashion, providing a few examples before the description.

Example 1: {5,6,7,…,100}

Initially find a function that maps integers (preferably starting from 0) to the elements above.

In this case it is easy to see that $f(n)=5+n$ does that job. Indeed $f(0)=5$, $f(1)=6$ etc. Now we are looking for the value of $k$ which is mapped to $100$. Equations are made for this exact task! We thus have: $$f(k)=100 \Rightarrow 5+k=100 \Rightarrow k=95.$$

Proceed by noticing that the list $\{5,6,7,…,100\}$ is the same as $\{f(0),f(1),…,f(95)\}$, which of course has the same number of elements as $\{0,1,…,95\}$ has. Therefore that sequence has 96 elements, and the counting problem is solved.

Example 2: {36,39,42,…,141}

Now this is certainly a little more trickier than before. But using the same technique, it is easy to see that a function that tracks the elements of this sequence is $f(n)=36+3\cdot n$. We are interested (again) in the $k$ which is mapped to $141$. Equations to the rescue!

$$f(k)=141 \Rightarrow 36+3k=141 \Rightarrow k=35 $$

Now beautifying our sequence, we can rewrite it as $\{ f(0),f(1),…,f(35) \}$ which has 36 elements.

Example 3: {87.3, 86.2, 85.1, …, -7.3}

This problem is used to show that the set of objects or numbers represented doesn’t really matter. You could enumerate pretty much anything as long as you can find a function that tracks the elements of the sequence and you can solve equations in this function.

$$f(n)=87.3 - 1.1 \cdot n$$

Finding the edge case $$f(k)=-7.3 \Rightarrow 87.3-1.1\cdot k=-7.3 \Rightarrow k = 86 $$

Thus, the number of elements is 87.

The Method, dissected

The method that was followed, briefly, is the following:

  1. Have a sequence $x_0,x_1,…,x_k$

  2. Find a function $f$ that describes the sequence. The function should be found relatively easy if it is clear to you how the sequence behaves. There are always infinite functions that will satisfy your sequence. Any of those will be good. Above I preferred functions that for $n=0$ they are equal with the first number of the sequence. Although that’s not required, it makes things simpler.

  3. Find the independent variables (the argument) for which your function gives the two edge values.

  4. get the result as: $\text{final}-\text{start}+1$

in step 3, finding an integer solution is required, otherwise the function will never reach the final number - in other words you function doesn’t really track your sequence.

Note that the cases we used are all linear. That’s not required, but it the common case. Any bijection in the required range will do. You should be careful in more involved cases, because having a non-bijective function can lead to double counting or missing numbers.

In a high-level description, the sequence of interest was condensed to just two parameters. The function, and the ending independent variable. The starting independent variable is also a parameter, but it is inferred from the function.

For instance in the third example we have:

$\{87.3,86.2,85.1,…,-7.3\} \equiv (f(n)=87.3-1.1 \cdot n, 35) $

You can view this as a way of compressing the sequence.
